# 데이터 소스 명세

모든 데이터는 무료 공공 API. 개인 단위 데이터는 존재하지 않으며 사용하지 않는다.
집계 통계로 합성 인구(synthetic population)를 생성하는 것이 방법론의 전제다.

## 1. 인구 구성 → 합성 인구 생성 재료
- 통계청 KOSIS / SGIS(통계지리정보서비스): 동 단위 성별·연령·가구 구성, 직업 분포
- 행정안전부 주민등록 인구통계: 동 단위, 월 갱신
- 용도: 시민 에이전트의 프로파일 분포(나이·직업·거주지) 결정

## 2. 이동 패턴 → 에이전트 동선 재료
- 서울시 생활인구 (열린데이터광장): 기지국 기반 시간대별·동 단위 체류 인구
- 대중교통 승하차 데이터: 정류장·역 단위 시간대별 이용량
- OpenStreetMap (OSMnx로 수집): 도로망 그래프, 경로 탐색
- 용도: 출퇴근 시간 분포, 유동 패턴 캘리브레이션

## 3. 상권·매장 → 시뮬레이션 세계 구성 + 검증 재료
- 소상공인시장진흥공단 상가(상권)정보 API: 전국 상가 위치·업종
- 서울시 상권분석서비스: 상권별 추정매출·점포수·유동인구
- 용도: (a) 가상 동네에 실제 매장 배치 (b) **백테스트 검증의 정답지**

### 구현 범위 노트 (2026-07-21 조사 후 확정)
Phase 1의 수집 스크립트는 **생활인구(행정동 단위)만 우선 구현**한다. 나머지 두 API는 백테스트·매장 배치 단계로 미룬다.
- **생활인구(행정동 단위)**: 서비스명 `SPOP_LOCAL_RESD_DONG` (data.seoul.go.kr Open API 탭에서 직접 확인). 성동구 자치구코드는 `11200`(자치구 단위 생활인구 라이브 데이터로 확인) — 행정동코드는 `11200`으로 시작하는 값을 클라이언트에서 필터링해 사용한다. 성수동 개별 행정동 4개(성수1가1동/1가2동/2가1동/2가3동)의 정확한 코드는 코드↔이름 매핑을 아직 확보하지 못해 합성 인구 생성 단계로 미룸.
- **상권분석서비스(추정매출)**: 서비스명 `VwsmTrdarSelngQq`. 상권코드로 직접 필터링하는 요청 파라미터가 없어 서울 전역(약 46만 행)을 페이징하며 상권코드→행정동 매핑(별도 "영역-상권" 데이터셋)까지 조인해야 정확한 지역 필터링이 가능 → 범위가 커서 이연.
- **상가(상권)정보**(data.go.kr): REST/Swagger 기반 API로, 정확한 요청 엔드포인트가 로그인 후 문서(또는 Swagger UI)에서만 확인 가능해 이번 조사에서 완전히 검증하지 못함(추정 엔드포인트: `apis.data.go.kr/B553077/api/open/sdsc/storeListInDong` — 확인 필요) → 이연.

## 인증키 신청처
- 공공데이터포털 data.go.kr → 상가(상권)정보 활용신청
- 서울 열린데이터광장 data.seoul.go.kr → 생활인구, 상권분석
- KOSIS 공유서비스 kosis.kr → 인증키 발급

## 주의사항
- 지역 격차: 서울은 생활인구+추정매출 공개로 데이터 품질 최상. 경기권은 빈약
  → 데모 지역을 서울(성수동)로 고정한 이유
- API 키는 .env로 관리, 절대 커밋 금지
- 수집 스크립트는 재실행 가능(idempotent)하게 작성: 같은 데이터 중복 적재 방지
