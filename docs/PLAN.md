**완료 기준: 시민들이 출퇴근하고 매장에 들르는 모습이 실시간으로 보인다

## Phase 3 — 지능층과 what-if 실험
- [x] LLM 의사결정층: 전환점 상황에만 경량 모델 호출, 프로파일 기반 프롬프트 (`engine/decision/`, Gemini — DECISIONS D10)
- [x] 비용 최적화: 유사 프로파일 캐싱 + 배칭 (`engine/decision/cache.py`, `decider.py`)
  - 실제 Gemini API로 검증 완료: 거리에 따라 전환 확률이 합리적으로 갈림(가까움 0.85 / 멀고 기존 매장 없음 0.3),
    동일 프로파일 재요청 시 LLM 재호출 없이 캐시 재사용 확인
- [x] what-if 주입: 지도에 신규 매장 배치 → 방문 수·이탈 출처·시간대 분포 집계
  (`engine/whatif/`, `POST /api/whatif`) — 단골 매장 개념 도입(DECISIONS D9 해소),
  하루 시작 시점 배치 LLM 판단, 응답 개수 불일치·503 과부하 방어(재시도) 모두 실제로
  겪어서 고침. 성수동 실좌표 7일 시뮬레이션(시민 1,000명) 실제 성공: 일평균 방문
  353.4명, 전환 380명, 이탈 출처(CU·버거점프·정이담·스타벅스 등)·시간대 분포(17~23시
  집중, 21시 피크) 모두 합리적으로 산출됨. 웹 UI(지도 클릭 → 폼 → 리포트 표시)는 아직 미착수
- [ ] 결과 대시보드: 핵심 지표, 이탈원 차트, 시간대 차트
